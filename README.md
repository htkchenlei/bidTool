# BidTool — 智能投标辅助工具

面向投标业务全流程的辅助工具，核心能力是基于大语言模型（AI）的智能分析。支持从公告网页抓取、投标文件（PDF/DOCX）解析，到关键字段自动提取、废标风险智能识别的一站式操作。

## ✨ 核心特性

| 特性 | 描述 |
|------|------|
| 🤖 **AI 智能分析** | 大模型自动提取项目关键字段（29项）、识别12类废标风险条款 |
| 🏆 **业绩管理** | 上传合同 PDF、多字段筛选、批量下载、PDF 水印 |
| 🌐 **网页公告抓取** | 输入公告URL，自动转换为 Markdown 文本供 AI 分析 |
| 📄 **多格式文件解析** | 支持 PDF、DOCX、图片（OCR）等格式的文本和表格提取 |
| 🔍 **多源交叉匹配** | 融合公告网页和投标文件内容进行分析，信息更全面 |
| ⚠️ **风险条款识别** | 自动识别 `★`、`*`、`▲` 等特殊标记条款及废标风险 |
| ✅ **人工复核** | 提取结果支持在线修改、确认，确保准确性 |
| 🎯 **多模型支持** | 支持 OpenAI、DeepSeek、千问、智谱等多种 AI 模型 |

## 🛠️ 技术栈

| 层次 | 技术 |
|------|------|
| **后端** | Python 3.8+, Flask 3.0+, Flask-CORS |
| **前端** | HTML5 + CSS3 + Vue 3（CDN，无需构建） |
| **数据存储** | JSON 文件（`data/` 目录） |
| **AI 集成** | OpenAI 兼容 API（支持多种模型） |
| **文档解析** | pdfplumber (PDF), python-docx (DOCX) |
| **PDF 水印** | PyPDF2 + reportlab（半透明对角线水印） |
| **网页处理** | markitdown (URL → Markdown) |
| **OCR（可选）** | pytesseract + Tesseract |

## 🚀 快速开始

### 环境要求

- Python 3.8+
- （可选）Tesseract OCR — 用于非视觉模型的图片识别

### 安装运行

**方式一：一键启动（Windows）**

```bash
双击 start.bat
```

**方式二：命令行**

```bash
# 安装依赖
pip install -r requirements.txt

# 启动服务
python backend/app.py
```

启动后浏览器访问 `http://127.0.0.1:5000`。

## 📖 功能模块详解

### 1. 工作台（Dashboard）

- 关键统计数据展示（项目数、文件数、风险数等）
- 即将到期证书提醒
- 快速操作入口

### 2. 项目管理（核心功能）

**一站式 AI 分析流程：**

1. **新建项目** → 填写项目基本信息和公告URL
2. **上传文件** → 支持 PDF、DOCX、图片等格式
3. **开始分析** → 系统自动执行：
   - Markitdown 抓取公告网页并转换为 Markdown
   - 解析所有上传文件，提取文本和表格
   - 拼接多源内容发送给大模型
   - AI 提取31项关键字段
   - AI 识别12类废标风险条款
4. **查看结果** → 在线复核、修改、确认
5. **导出报告** → 生成分析报告

**AI 提取的29项字段：**

| 类别 | 字段 |
|------|------|
| 基础信息 | 项目名称、项目编号、标段号、招标方式、采购类型、项目地点 |
| 机构与联系人 | 招标人、代理机构、双方联系人及电话、电子邮箱 |
| 金额信息 | 预算金额、最高限价、控制价、投标保证金、履约保证金 |
| 时间信息 | 公告发布、报名起止、答疑截止、保证金截止、投标截止、开标时间 |
| 地点信息 | 报名地点、递交地点、开标地点 |
| 行业分类 | 所属行业 |

**AI 识别的12类风险：**

1. 明确废标/否决投标
2. 资格审查不通过
3. 符合性审查不通过
4. 特殊标记实质性条款（★/*/▲）
5. 不允许负偏离
6. 签字盖章要求
7. 密封和递交要求
8. 投标保证金要求
9. 报价和最高限价要求
10. 服务期限或交付期要求
11. 人员配置或项目负责人要求
12. 业绩证明或案例要求

### 3. 文件管理

- 文件上传、下载、删除
- 文件夹组织管理
- 文件类型自动识别

### 4. 资质证书管理

- 企业资质全生命周期管理
- AI OCR 自动识别证书关键信息
- 支持分类管理、搜索筛选

### 5. 区域查询

- 按省份/城市查询招标信息
- 多条件筛选、分页浏览

### 6. 报价计算

- 6类成本项内联编辑
- 3种报价策略（激进/平衡/保守）
- 利润率、折扣率、税率灵活调整

### 7. 业绩管理

- 新建业绩：填写项目名称、乙方名称、日期、金额等信息，上传合同 PDF
- 编辑/删除现有业绩
- 多字段筛选查询：关键词、乙方名称、日期范围、分类、是否有文件
- 批量选择下载：勾选多份业绩打包下载为 ZIP
- PDF 水印：下载时可添加自定义水印文字，自动为 PDF 每页添加半透明对角线水印

### 8. 系统设置

- 多 AI 模型配置（OpenAI/DeepSeek/千问/智谱/自定义）
- 一键测试模型连接
- 视觉模型支持图片识别

## 🔧 AI 模型配置

在系统设置页面或 `data/config.json` 中配置：

```json
{
  "models": [
    { "name": "OpenAI GPT", "model": "gpt-4o", "supports_vision": true },
    { "name": "DeepSeek", "model": "deepseek-v4-flash", "supports_vision": false },
    { "name": "通义千问", "model": "qwen-max", "supports_vision": false },
    { "name": "智谱 GLM", "model": "glm-4-flash", "supports_vision": false },
    { "name": "自定义模型", "model": "", "supports_vision": false }
  ],
  "active_model": "通义千问"
}
```

## ⚠️ 注意事项

1. **数据目录**：`data/` 目录存放运行时数据（项目文件、解析结果、配置等），已在 `.gitignore` 中忽略
2. **扫描版 PDF**：对于扫描版（图片型）PDF，系统无法直接提取文本层，建议使用文字版 PDF 或 DOCX 文件
3. **网络要求**：AI 分析需要联网调用大模型 API
4. **文件大小**：单个文件建议不超过 50MB
